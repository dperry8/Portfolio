About

I am a graduating senior at The Ohio State University with a major in Computer Science and Engineering specializing in Computer Graphics and Game Design and a minor in Game Studies. I have been taught the practices of game development, the fundamentals of building structural code from the ground up, and the importance of games socially, culturally, and impactfully. I've been developing video games on my own time for 3 years and continuing to grow proficient in Unity. Outside of technical skills, I have led many projects, both creatively and managerially. I have also had real world experience as a software developer at Great American Insurance, being part of their API team for a summer.

I am always adaptable to any team I work with and what roles need to be filled. I am willing to face any challenge, and although I may not know the solution immediately, I work relentlessly until I can solve it. I continue to practice and sharpen my skills game development, version control, and project management. and I am currently looking for work with an indie game studio or entry level position at a company, both where I can expand my knowledge of the industry and offer my skills. I am passionate about all the work I do. Being a game developer is a life long dream, and I plan to accomplish it.

Battle of the Bands

Description

Battle of the Bands is a chess like PvP local multiplayer game. This is the first game I ever developed. I plan on returning to it in the future to improve the overall quality and add more more features with my new skills.

Takeaways

The code structure of the bands, members, and their abilities is well organized and reusable to easily create more. The areas to improve on is the game flow. There is only one Monobehaviour class controlling the entire game. This caused for some buggy moments and difficult debugging.

What I Learned

Developed in Unity. Skills used were 2D development, working with a canvas, UI, raycasting, and basic inputs (Old Input System). In this project, I learned the fundamentals of project management and code structure.

Euler Ball

Description

Euler Ball is a short game based around basic transformations. You move the map to try and get the ball from the start to the goal. This was completed in 24 hours, a personal game jam I set aside for myself, hence the shorter length and polish of the game.

Takeaways

I strengthened my skills on how to get a prototype of gameplay up quickly. This is useful when a feature is wanted to be seen quick, disregarding polish, so you know if it's a feature to be included.

What I Learned

Developed in Unity. Skills used were basic inputs (New Input System), canvas, transformations, and basic prefab construction.
